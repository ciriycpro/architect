# 1. N8N как платформа разработки агентов

Date: 2026-04-27

## Status

Accepted

## Context

Нужна платформа для автоматизации документооборота ИП Таирова: получение документов по почте/Drive, OCR, извлечение полей, запись в реестр Google Sheets, алерты в мессенджеры.

Кандидаты: чистый Python (LangGraph), N8N (low-code), Make.com (no-code), KAMF (свой фреймворк).

## Decision

Выбран **N8N** как основная платформа.

## Consequences

**Плюсы:**
- Быстрый старт (готовые ноды для IMAP, Drive, Sheets, HTTP)
- Визуальная отладка через UI
- Низкий порог входа — можно показать заказчику логику
- Уже развёрнут на сервере coo (Docker, порт 5678)

**Минусы:**
- Тяжёлый по RAM (~500 MB на инстанс)
- Сложно версионировать (workflow живут в БД, не в файлах)
- IMAP-нода глючит на больших вложениях (см. DEC-005)
