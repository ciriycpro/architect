# 3. Документовед на 39 нод (отменено)

Date: 2026-04-29

## Status

Cancelled — superseded by [DEC-004](0004-simplify-to-12-nodes.md)

## Context

Первая итерация Документоведа собрана на 39 нод:
- 3 триггера (IMAP + 2 Drive Trigger + Webhook)
- 5-ступенчатый OCR-каскад (cloudflare-ai → Qwen3-VL → Qwen2.5-VL → Mistral)
- 4 IF-ноды между ступенями
- Code-нода с pdf-parse/mammoth (require!)
- LLM Extract DeepSeek
- Code валидация
- IF "не подписан"
- Switch по 5 типам документов
- 5 отдельных Sheets

## Decision

Признано переусложнением. Отменено.

## Consequences

**Что не так:**
- 39 нод нереально дебажить через UI
- Каскад из 5 OCR — преждевременная оптимизация (на MVP хватает одного)
- Логика "не подписан" — это задача Контроллера, не Документоведа
- 5 разных Sheets вместо одного с типом-колонкой — переусложнение

**Урок:** на MVP не закладывать fallback'и до того, как появится статистика реальных ошибок.

**Реализационный артефакт (зафиксировано 09.05.2026 при сверке coo).** В БД N8N на coo сохранён workflow `Документовед` (id `2b0af1a8c92c424f`, 39 нод, active=0, last update 29.04.2026 01:04 UTC). Состав соответствует описанию выше: 5-ступенчатый OCR-каскад (pdf-parse/mammoth → cloudflare-ai → Qwen3-VL → Qwen 2.5 VL → Mistral OCR), DeepSeek V3 LLM Extract, IF "не подписан" с алертом через HTTP→Коллер, Switch по 5 типам с записью в 5 разных Sheets (Договоры, Счета, Акты, Выписки, Запросы). Workflow деактивирован при принятии DEC-004, но не удалён. Решение по дальнейшей судьбе артефакта — см. [DEC-006](0006-reality-check-09052026.md).
