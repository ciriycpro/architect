# 4. Упрощение Документоведа до 12 нод

Date: 2026-04-30

## Status

Accepted (decision) — Not implemented as of 09.05.2026 — **Superseded by [DEC-008](0008-parser-stack.md) on 12.05.2026** (заменён горизонтальным стеком mail-stack вместо монолитного N8N workflow)

## Context

После анализа Франкенштейна-1 (39 нод, см. DEC-003) решено упростить до минимально жизнеспособной версии.

## Decision

**Документовед = 12 нод:**

1. Drive Trigger (от Полины) — единственный источник
2. Drive: download
3. HTTP Request → Qwen3-VL 235B (один OCR, без каскада)
4. HTTP Request → DeepSeek V3 (LLM extract JSON)
5. Code: валидация полей
6. IF: распознали корректно?
   - нет → Sheets "Ошибки" + Drive Move в "Требует проверки" + HTTP→Caller
   - да →
7. Switch: тип документа (5 веток)
8. Sheets: Append Row
9. Drive Move в Обработанные/<тип>

## Consequences

**Плюсы:**
- В 3 раза проще, реально дебажится
- Один OCR — одна точка отказа, понятная для логирования
- Логика "не подписан" вынесена в Контроллер (как и должно быть)

**Минусы:**
- Один OCR без fallback — если Qwen упадёт, пайплайн встанет (приемлемо для MVP)
- Drive Trigger от Полины как единственный источник — отвалился IMAP-канал от Таирова

**Открытый вопрос:** как вернуть IMAP-канал? — решено в DEC-005.

**Статус реализации (зафиксировано 09.05.2026 при сверке coo).** В БД N8N на coo нет workflow, реализующего описанный 12-нодовый пайплайн. Документовед v2 как описан в этом ADR — не собран ни одной нодой. На coo существуют только:
- Inactive `Документовед` (id `2b0af1a8c92c424f`, 39 нод) — реализация отменённого DEC-003.
- Active `Полигон` (id `1632604d64e14e20`, 14 нод, last update 02.05.2026 23:36 UTC) — выполняет другую функцию: приём писем по IMAP с раскладкой вложений по папкам отправителей в Google Drive и логированием в Sheets. OCR / LLM Extract / Switch по типам в Полигоне отсутствуют.

История Полигона: задумывался как чистый стенд, на который инкрементально добавлялись бы ноды до полного Документоведа. После инцидента в ночь со 02 на 03 мая 2026 (Полигон не видел писем в реальном времени, IMAP-нода N8N работала нестабильно) — направление развития сменилось на DEC-005 (вынесение приёма почты в отдельный микросервис). Полигон остался как полу-заглушка.

Решение по дальнейшей реализации Документоведа v2 — см. [DEC-006](0006-reality-check-09052026.md).
