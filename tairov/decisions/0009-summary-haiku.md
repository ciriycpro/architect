# 9. Summary-service: Claude Haiku 4.5 для ежедневного дайджеста

Date: 2026-05-12

## Status

Accepted — **Implemented on 13.05.2026** (production via systemd, Memory 30.6 МБ, port 8768)

## Context

В архитектуре `mail-stack/` v1 финальный этап — формирование ежедневного дайджеста писем для отправки Таирову (см. [DEC-010](0010-n8n-email-digest-v1.md)). Это **витринный артефакт продукта**: каждое утро Таиров видит текст этого саммари в Telegram. Качество имеет значение для удержания клиента.

Задача — текстовая, не vision: вход — массив текстов всех писем за день (≤50 шт × ~500 символов = ~25K символов ≈ ~8K токенов), выход — структурированное саммари ~500-1000 токенов на русском языке.

Объём вызовов: 1 дайджест/день × 30 дней = 30 LLM-вызовов в месяц.

Рассматривались четыре кандидата на OpenRouter:

| Модель | Цена in / out | Сильные стороны | Слабые |
|---|---|---|---|
| **DeepSeek V3** | $0.27 / $1.10 | Дёшево, прилично на русском, длинный контекст | Суховатый стиль, иногда холодное саммари |
| **Qwen 2.5 72B** | $0.30 / $0.40 | Дёшево, фактологический | Может пропустить нюансы |
| **Claude Haiku 4.5** | $0.80 / $4.00 | Лучший тон на русском, структурированный markdown, кириллица идеально | Дороже в 3-5 раз |
| **GPT-4o-mini** | $0.15 / $0.60 | Самый дешёвый | Средне на русском, иногда сбой кодировки |

**Экономика на объёме 30 вызовов/мес:**
- DeepSeek V3: ~5 руб/мес
- Qwen 2.5 72B: ~6 руб/мес
- GPT-4o-mini: ~4 руб/мес
- **Claude Haiku 4.5: ~25 руб/мес**

Разница между самым дешёвым и Haiku — 20 руб/мес = 240 руб/год. На чеке Таирова 50К/мес — это **0.05% выручки**.

## Decision

Использовать **Claude Haiku 4.5** через OpenRouter для summary-service.

### Аргументы

1. **Дайджест — витринный артефакт.** Это единственный текст, который Таиров видит каждое утро. Тон, структура, отсутствие галлюцинаций — критичны для восприятия продукта.

2. **Разница в цене незначима в абсолюте.** 20 руб/мес на одном клиенте — не та статья расходов, на которой имеет смысл экономить ценой качества.

3. **Принцип «всё должно окупаться» — соблюдён.** 25 руб/мес vs чек 50К/мес = 0.05%. При масштабировании на 100 клиентов: 2500 руб/мес vs ~5М выручки = всё те же 0.05%. Линейно масштабируется без потери маржинальности.

4. **Лучший тон на русском среди API-моделей.** Claude известен этим объективно — DeepSeek и Qwen генерируют более «суховатый», википедийный текст. Для Таирова (бизнес-клиент, ожидает понятную человеческую речь) — Haiku предпочтительнее.

5. **Структурированный markdown консистентно.** Haiku надёжно соблюдает заданную структуру ответа (заголовки, списки, выделения). DeepSeek иногда ломает разметку.

### Промпт-контракт summary-service

Endpoint: `POST /summary`

```json
Request:
{
  "period": "2026-05-12",
  "messages": [
    {
      "from": "Иван Иванов <ivan@x.ru>",
      "subject": "Счёт за апрель",
      "body_text": "...",
      "attachment_names": ["счёт.pdf"],
      "parsed_attachments": [
        {"text": "...", "method": "pymupdf", "format": "pdf-text"}
      ]
    },
    ...
  ]
}

Response:
{
  "summary_markdown": "## Дайджест за 12 мая\n\n**Всего писем:** 8\n...",
  "summary_telegram": "Дайджест за 12 мая\nВсего писем: 8\n...",
  "tokens_in": 7234,
  "tokens_out": 892,
  "cost_usd": 0.008,
  "model": "anthropic/claude-haiku-4-5"
}
```

Два формата вывода — markdown (для Google Sheets «Дайджест») и plain-text без разметки (для Telegram, где markdown не везде корректно отрендерится).

### Структура промпта

#### Промпт v1 (изначальный, чисто структурный)

```
Ты — ассистент бухгалтера ИП Таирова. Тебе на вход дан массив писем за день.
Твоя задача — сделать краткий дайджест:

1. Общее количество писем и распределение по отправителям
2. Ключевые требующие действий: счета к оплате, договоры к подписанию, требования из ФНС, исковые материалы
3. Информационные письма (без требуемого действия): уведомления, рассылки
4. Письма с вложениями — что за документы, тип
5. Спам / нерелевантное — отдельным блоком, кратко

Ответь в JSON:
{
  "summary_markdown": "<markdown с заголовками>",
  "summary_telegram": "<plain-text для Telegram>"
}

Не выдумывай факты. Если в письме нет ясности — пиши «требует уточнения».
```

#### Промпт v2 (актуальный — живой тон, без канцелярита)

Реализован 13.05.2026 на этапе smoke-тестирования. Замечено: v1 даёт корректный, но сухой канцелярский тон («Дайджест», «Сводка», «Перечень»). Для использования помощника пользователем — это PR-офисный язык. Переработан под **живой доклад от помощника** с применением best practices индустрии (Anthropic XML-tags, few-shot example, action verbs, plain terms, length constraints).

Полный промпт хранится в `/opt/mail-stack/summary-service/server.py` (константа `SYSTEM_PROMPT`).

Ключевые изменения:
- **Role:** «персональный AI-ассистент Артёма Якшина, который сам прошёл по почте». Не «ассистент бухгалтера».
- **Tone:** деловой друг, не казённый. Без слов «дайджест», «сводка», «перечень», «раздел», «блок».
- **Приветствие варьируется** — 4 примера в промпте, инструкция рандомизировать.
- **Действия описывать живым языком**: «ФНС хочет выписки до 20 мая» вместо «Требование ФНС о предоставлении документов».
- **Числа округлять для устной речи**: «45000 → 45к», «1 500 000 → 1.5 миллиона».
- **Обращение** — «Привет, Артём» или просто «Привет», не «Уважаемый».
- **Описание изображений напрямую без эвфемизмов**: «портрет женщины», «пейзаж с морем».
- **XML-теги** для структурирования промпта (`<role>`, `<task>`, `<structure>`, `<format>`, `<example>`, `<rules>`) — Anthropic-семейство лучше парсит XML чем markdown.
- **Few-shot example** — два примера: на 3 письма с реальными данными Таирова + на спокойный день.
- **Length constraint** — summary_telegram строго не более 1000 символов с явной логикой приоритизации (критичное → важное → информационное → спам).
- **Action verbs** — «Извлеки», «Опиши», «Перечисли», «Группируй».
- **Эмодзи как маркеры разделов** (⚠️ 📎 📌 🗑) **БЕЗ слов** «Сводка», «Раздел».

Зафиксированный пример вывода (на 2 письмах Контур.Экстерн от ФНС):
```
Привет, в почте за неделю два уведомления от Контура по АСВИТС.

⚠️ Странная ситуация: ФНС 7710 отправила решение о выездной проверке (12 мая, 14:13), 
а через 5 минут — решение о приостановлении этой же проверки. Надо зайти в Экстерн 
и разобраться, что произошло. Может быть ошибка, может быть реальное решение.

Остальное спокойно.
```

332 символа, цена $0.0049 ≈ 0.37 руб.

### Fallback

Если OpenRouter недоступен или Haiku rate-limit'нул → fallback на **DeepSeek-chat** (актуальный alias OpenRouter, ведёт на DeepSeek V3.1+). Это редкий путь (рейт-лимиты Haiku на нашем объёме не достижимы), но fallback должен быть. Зафиксировано в коде summary-service как try/except.

### Защита от ребрендинга моделей

OpenRouter поддерживает алиасы `~anthropic/claude-haiku-latest`, `~anthropic/claude-sonnet-latest`, `~anthropic/claude-opus-latest`. Эти алиасы автоматически ведут на последнюю версию модели семейства. Сейчас в env-файле primary — конкретная версия `anthropic/claude-haiku-4.5`. Это компромисс: предсказуемость стоимости и поведения vs автоматическая защита от ребрендинга. На v2 — возможен переход на `~anthropic/claude-haiku-latest` если выработка темпа Anthropic-релизов потребует ручных правок чаще раза в квартал.

### Эпизод с экономикой выбора модели

13.05.2026 в процессе smoke-тестирования возник вопрос: «Haiku 4.5 = 0.44 руб/дайджест — это оверпрайс?». Рассмотрены альтернативы:

| Модель | Цена/дайджест | × к Haiku | На 100 клиентов/мес |
|---|---|---|---|
| **Claude Haiku 4.5** (текущая) | 0.37-0.44 руб | 1× | ~1300 руб |
| DeepSeek-chat (v3.1) | ~0.13 руб | 0.30× | ~400 руб |
| Qwen-2.5-72B / GLM-4.6 | ~0.07 руб | 0.16× | ~250 руб |

**Решение: остаёмся на Haiku 4.5 как primary.** Аргументы:
- Стоимость на текущем объёме (1 клиент) — копейки, разница 0.30 руб/день незначительна
- Качество русского у Haiku 4.5 заметно выше DeepSeek, особенно для **живого разговорного тона** (промпт v2 рассчитан именно на это)
- Стратегически: первое впечатление Таирова важнее экономии 30 коп/день. Опуститься в качестве потом легко (одна строка в env-файле), подняться обратно — теряем неделю на доводку промпта под другую модель
- На v2/v3 при росте масштаба — пересмотреть. Зафиксировано как открытый вопрос.

## Consequences

**Плюсы:**
- Лучшее качество текста дайджеста среди реалистичных по цене вариантов
- Минимизация риска плохого впечатления Таирова
- Линейное масштабирование стоимости (без скачков при росте до 100 клиентов)
- Fallback на DeepSeek закрывает риск недоступности

**Минусы:**
- 5× дороже самого дешёвого варианта в абсолюте (но 20 руб/мес — pet money)
- Зависимость от Anthropic API (через OpenRouter) — если OpenRouter упадёт, fallback ещё одной модели тоже через OpenRouter. Митигация v2: добавить прямой Anthropic API endpoint как третий уровень fallback
- Лицензионных рисков (как с AGPL у PyMuPDF) — нет, это commercial API

**Открытые вопросы для v2:**
- Когда переключиться на Sonnet 4.5 — если дайджесты станут более сложными (например, начнут включать рекомендации к действию)
- Когда добавить персонализацию дайджеста (тон, ключевые слова конкретного клиента) — после набора статистики предпочтений Таирова
- **Переход на DeepSeek-chat (V3.1) как primary при росте масштаба.** Экономия ~3× ($0.30/мес vs $1.30/мес на клиента). Trade-off: качество русского языка у DeepSeek заметно ниже Haiku, особенно для живого разговорного тона. Требует доводки промпта v2 под другую модель + A/B-тест с Таировым. Триггер пересмотра — 50+ клиентов на тарифе.
- **Алиас `~anthropic/claude-haiku-latest`** для автоматической защиты от ребрендинга моделей Anthropic. Trade-off: предсказуемость стоимости vs автоматическое следование релизам. Триггер — если Anthropic переименовывает модели чаще раза в квартал.
- **DEC-015 «Persistent prompt rules per user»** — Telegram-команды `/digest_add`, `/digest_remove`, `/digest_tone` для prompt evolution через диалог. Пользователь учит систему за месяц-два до своих предпочтений без правки кода. Архитектура: state-service хранит «пользовательские правила к промпту», summary-service дёргает state-service перед каждым summary-вызовом и дополняет базовый промпт правилами. ~200 строк кода. Кандидат на отдельный ADR на v2.
- **Разделение JSON-вывода на два последовательных вызова** (markdown отдельно + telegram отдельно). Сейчас два формата в одном вызове, что снижает качество (best practice индустрии: «один формат за один вызов»). Trade-off: ×2 стоимость и +1 LLM-вызов. На v2 при росте качества как приоритета.
- **Промпт-эволюция через A/B-тестирование.** На v3 рассмотреть автоматическое A/B-тестирование вариаций промпта на одних и тех же дайджестах с метрикой удовлетворённости пользователя (например, кнопка ⭐ в Telegram под каждым дайджестом → собирает feedback → подбирает оптимальный промпт).

## Implementation Notes (13.05.2026)

Реализован за ~1 час от создания каталога до production. Самый быстрый сервис в стеке — нет тяжёлых зависимостей, нет компиляции, нет специализированной логики (как в parser-service L1-L14). Чистая работа с API и промптом.

### Что сделано

- **Код:** `/opt/mail-stack/summary-service/server.py` (~280 строк, 9 функций включая endpoint)
- **Зависимости:** 4 базовых пакета (FastAPI 0.136.1, uvicorn 0.46.0, pydantic 2.13.4, httpx 0.28.1) + 13 транзитивных = 17 пакетов в pinned `requirements.txt`
- **Env-конфиг:** `/etc/mail-stack/summary-service.env` chmod 600 root:root, OPENROUTER_API_KEY + 5 параметров (модели, temperature, max_tokens, timeout)
- **Systemd-юнит:** `/etc/systemd/system/summary-service.service` с UMask=0022, EnvironmentFile, Restart=always
- **Docker-образ:** `summary-service:test` собран за 3 секунды (python:3.11-slim + 4 чистых wheels)
- **Memory в production:** 30.6 МБ — рекорд стека (меньше mail-service)

### Архитектурное наблюдение

`summary-service` — это **stateless transformer**. Никакого диска, никакой БД, никаких side-effects. Идеальный кандидат для горизонтального масштабирования: можно запустить N инстансов за load-balancer для роста до 100+ клиентов. Это **противоположность** parser-service (тяжёлая компиляция, специфический wheel, лимит памяти).
